## 1. What is a PWA?

A **Progressive Web App (PWA)** is a website that behaves like a native mobile/desktop app.

Think of it this way:
- You visit `finsense.com` in Chrome on your phone
- Chrome shows a popup: **"Add FinSense to Home Screen"**
- You tap it → FinSense gets installed like an app
- You can open it from your home screen, it has its own icon, no browser bar
- It even **works offline** (shows cached data or a nice offline page)

### PWA vs Normal Website vs Native App

| Feature | Normal Website | PWA | Native App |
|---|---|---|---|
| Works Offline | ❌ | ✅ | ✅ |
| Installable | ❌ | ✅ | ✅ |
| Push Notifications | ❌ | ✅ | ✅ |
| App Store needed | ❌ | ❌ | ✅ |
| Single Codebase | ✅ | ✅ | ❌ |
| Fast Loading (cached) | ❌ | ✅ | ✅ |

## 2. Why Add PWA to FinSense?

FinSense is a **financial management app** — users check it frequently. PWA gives:

- ✅ **Fast loading** — Assets cached by Service Worker, loads instantly even on slow internet
- ✅ **Offline support** — Users can view their last loaded dashboard data without internet
- ✅ **Installable** — Users can add FinSense to their phone/desktop home screen
- ✅ **Better UX** — Fullscreen mode (no browser address bar), feels like a real app
- ✅ **Push Notifications** — (future) Notify users about budget limits, transaction alerts

## 3. How PWA Works (Core Concepts)

Understanding these 3 things is key:

### 3.1 Service Worker
- A JavaScript file that runs **in the background** (separate from the webpage)
- Acts as a **proxy/middleman** between your app and the network
- Can **cache files** (HTML, CSS, JS, images, API responses)
- Can serve cached content when **offline**
- Automatically generated by `next-pwa` — you usually don't write it manually

```
Browser Request → Service Worker → (Cache or Network) → Response
```

### 3.2 Web App Manifest (`manifest.json`)
- A JSON file that tells the browser **how to "install"** your app
- Defines: app name, icons, theme color, start URL, display mode (fullscreen/standalone)
- Located at `/public/manifest.json`
- Linked in your HTML `<head>` tag

### 3.3 HTTPS Requirement
- PWA **requires HTTPS** in production (Service Workers only work on HTTPS)
- `localhost` is the only exception (for development)
- Your existing Nginx + SSL setup already handles this ✅

### Step 3: Generate App Icons

You need icons in multiple sizes. Here's how:

#### Option A: Use a free online tool (Recommended for beginners)
1. Go to [https://www.pwabuilder.com/imageGenerator](https://www.pwabuilder.com/imageGenerator)
2. Upload your logo: `/public/icon-source.png` (you already have this!)
3. Download the generated icon pack
4. Create a folder `/public/icons/`
5. Place all generated PNG icons inside `/public/icons/`

#### Option B: Use Sharp locally (via a script)

Create a file `scripts/generate-icons.mjs`:

```js
import sharp from 'sharp';
import path from 'path';
import fs from 'fs';

const sizes = [72, 96, 128, 144, 152, 192, 384, 512];
const inputFile = path.resolve('public/icon-source.png');
const outputDir = path.resolve('public/icons');

if (!fs.existsSync(outputDir)) {
  fs.mkdirSync(outputDir, { recursive: true });
}

for (const size of sizes) {
  await sharp(inputFile)
    .resize(size, size)
    .toFile(path.join(outputDir, `icon-${size}x${size}.png`));
  console.log(`✅ Generated icon-${size}x${size}.png`);
}
```

Then run it once:

```bash
node scripts/generate-icons.mjs
```

> You already have `sharp` installed as a dev dependency, so this will work!

### Step 4: Configure next-pwa in `next.config.ts`

Replace your current `next.config.ts` with this:

```typescript
import type { NextConfig } from "next";
import withPWAInit from "@ducanh2912/next-pwa";

const withPWA = withPWAInit({
  dest: "public",              // Service worker files go into /public
  cacheOnFrontendNav: true,    // Cache pages when user navigates (SPA-style)
  aggressiveFrontEndNavCaching: true,  // Aggressively cache front-end navigation
  reloadOnOnline: true,        // Reload page when user comes back online
  swcMinify: true,             // Use SWC minifier for smaller SW file
  disable: process.env.NODE_ENV === "development",  // Disable SW in dev (avoids confusion)
  workboxOptions: {
    disableDevLogs: true,      // Clean console output
  },
});

const nextConfig: NextConfig = {
  output: "standalone",        // Keep your existing standalone output
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "res.cloudinary.com",
      },
    ],
  },
};

export default withPWA(nextConfig);
```

> **Why `disable: process.env.NODE_ENV === "development"`?**
> During development (`npm run dev`), Service Workers can cause confusing caching bugs. We disable it in dev and only enable it in production.

#### Create `src/components/AuthGuard.tsx`

Since we removed `"use client"` from layout, move the auth logic to a separate client component:

```tsx
"use client";

import { useEffect } from "react";
import { useRouter, usePathname } from "next/navigation";

const PUBLIC_ROUTES = ["/login", "/signup", "/forgot-password"];

export default function AuthGuard({ children }: { children: React.ReactNode }) {
  const router = useRouter();
  const pathname = usePathname();

  useEffect(() => {
    const token = localStorage.getItem("token");
    const isPublicRoute = PUBLIC_ROUTES.some((route) =>
      pathname.startsWith(route)
    );

    if (!token && !isPublicRoute) {
      router.push("/login");
    }
  }, [router, pathname]);

  return <>{children}</>;
}
```

> **Why this matters for PWA?**
> The root layout must be a **Server Component** so Next.js can properly inject `<head>` tags (manifest link, theme-color meta tags). These head tags are **required** for PWA installability.

### Step 6: Create a Custom Service Worker (Optional)

By default, `next-pwa` generates a service worker automatically. But you can add custom behavior.

Create `/public/sw-custom.js` (optional):

```javascript
// This file is optional — only add if you need custom caching behavior

// Example: Cache API responses for offline support
self.addEventListener('fetch', (event) => {
  const url = new URL(event.request.url);

  // Cache only GET requests to your own API
  if (url.pathname.startsWith('/api/') && event.request.method === 'GET') {
    event.respondWith(
      caches.open('api-cache').then(async (cache) => {
        try {
          const response = await fetch(event.request);
          cache.put(event.request, response.clone());
          return response;
        } catch {
          // Offline: return cached version
          return cache.match(event.request);
        }
      })
    );
  }
});
```

### Step 8: Add Install Prompt UI

When users can install your app, show them a nice banner. Create `/src/components/PWAInstallBanner.tsx`:

```tsx
"use client";

import { useEffect, useState } from "react";

interface BeforeInstallPromptEvent extends Event {
  prompt: () => Promise<void>;
  userChoice: Promise<{ outcome: "accepted" | "dismissed" }>;
}

export default function PWAInstallBanner() {
  const [deferredPrompt, setDeferredPrompt] =
    useState<BeforeInstallPromptEvent | null>(null);
  const [showBanner, setShowBanner] = useState(false);

  useEffect(() => {
    const handler = (e: Event) => {
      e.preventDefault();
      setDeferredPrompt(e as BeforeInstallPromptEvent);
      setShowBanner(true);
    };

    window.addEventListener("beforeinstallprompt", handler);
    return () => window.removeEventListener("beforeinstallprompt", handler);
  }, []);

  const handleInstall = async () => {
    if (!deferredPrompt) return;
    await deferredPrompt.prompt();
    const { outcome } = await deferredPrompt.userChoice;
    if (outcome === "accepted") {
      setShowBanner(false);
    }
    setDeferredPrompt(null);
  };

  if (!showBanner) return null;

  return (
    <div
      style={{
        position: "fixed",
        bottom: "1rem",
        left: "50%",
        transform: "translateX(-50%)",
        background: "linear-gradient(135deg, #6366f1, #8b5cf6)",
        color: "white",
        borderRadius: "1rem",
        padding: "1rem 1.5rem",
        display: "flex",
        alignItems: "center",
        gap: "1rem",
        boxShadow: "0 8px 32px rgba(99, 102, 241, 0.4)",
        zIndex: 9999,
        maxWidth: "400px",
        width: "90%",
      }}
    >
      <span style={{ fontSize: "2rem" }}>📲</span>
      <div style={{ flex: 1 }}>
        <div style={{ fontWeight: "bold" }}>Install FinSense</div>
        <div style={{ fontSize: "0.85rem", opacity: 0.85 }}>
          Add to home screen for the best experience
        </div>
      </div>
      <div style={{ display: "flex", gap: "0.5rem" }}>
        <button
          onClick={() => setShowBanner(false)}
          style={{
            background: "rgba(255,255,255,0.2)",
            border: "none",
            color: "white",
            padding: "0.4rem 0.8rem",
            borderRadius: "0.5rem",
            cursor: "pointer",
          }}
        >
          Later
        </button>
        <button
          onClick={handleInstall}
          style={{
            background: "white",
            color: "#6366f1",
            border: "none",
            padding: "0.4rem 0.8rem",
            borderRadius: "0.5rem",
            fontWeight: "bold",
            cursor: "pointer",
          }}
        >
          Install
        </button>
      </div>
    </div>
  );
}
```

Then add it to your `layout.tsx` inside `<AuthGuard>`:

```tsx

  <Header />
  <main>{children}</main>

  <PWAInstallBanner />   {/* ← Add this line */}

```

## 8. Common Errors & Fixes

### ❌ Service Worker not registering
**Cause:** Still running `npm run dev` (SW disabled in dev)
**Fix:** Run `npm run build && npm start` to test with real SW

### ❌ Manifest not found (404)
**Cause:** `manifest.json` not in `/public/` or wrong path
**Fix:** Ensure file is at `/public/manifest.json` (accessible at `http://localhost:3000/manifest.json`)

### ❌ Icons not loading
**Cause:** Wrong path or icons not generated
**Fix:** Check `/public/icons/` folder exists with all PNG files

### ❌ `"use client"` error in layout
**Cause:** Root layout is a client component — can't export `metadata`
**Fix:** Remove `"use client"` from `layout.tsx`, move client logic to `AuthGuard.tsx`

### ❌ `Cannot read properties of undefined (reading 'prompt')`
**Cause:** `beforeinstallprompt` event fired before component mounts
**Fix:** The `useEffect` in `PWAInstallBanner` handles this — ensure you're using the component as shown

### ❌ PWA not installable on iOS Safari
**Cause:** iOS Safari uses a different mechanism (no `beforeinstallprompt`)
**Fix:** Add instructions manually — iOS users: Safari → Share → "Add to Home Screen"
You can detect iOS and show a custom tooltip:
```tsx
const isIOS = /iPad|iPhone|iPod/.test(navigator.userAgent);
```

### ❌ Old cache served after update
**Cause:** Service Worker caching old version
**Fix:** `next-pwa` auto-handles this with cache versioning. On deploy, users get the new version after the SW updates (within 24h or on next visit)

## 9. How It Works in Production (Docker)

Your existing Docker + Nginx setup is already compatible! Just make sure:

1. **Build the Docker image** after adding PWA config:
```bash
docker build -t finsense:latest .
```

2. **HTTPS is configured** (already done with Nginx + SSL) — PWA requires HTTPS ✅

3. **After `npm run build`**, check that these files exist:
   - `/public/sw.js` — the generated service worker
   - `/public/workbox-*.js` — workbox runtime files

4. **These files are automatically included** in the `standalone` output since they're in `/public/`.

5. **Nginx note:** Make sure Nginx doesn't cache `sw.js` aggressively. Add this to your nginx config:
```nginx
location = /sw.js {
    add_header Cache-Control "no-cache, no-store, must-revalidate";
    add_header Pragma "no-cache";
    expires 0;
}
```

> This is important! If Nginx caches `sw.js`, users won't get app updates.
